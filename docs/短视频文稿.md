### 操作演示（0:50-2:30）— 四步实操

#### 第一步：认证（0:50-1:15）

**画面**：浏览器打开 index.html → 点击认证区域

**旁白/字幕**：

> 第一步：配置认证。
> 
> 推荐用 OpenAPI 方式——
> 打开 ima.qq.com/agent-interface，
> 复制 Client ID 和 API Key，
> 粘贴到这里，点保存。
> 
> **【画面操作实录】**
> - 打开 ima.qq.com/agent-interface 页面
> - 复制 Client ID
> - 回到工具界面粘贴
> - 复制 API Key
> - 粘贴并点击"保存"
> - 步骤 1 的数字徽标变为绿色 ✓
> 
> 一次填写，长期有效。比手动抓 Token 方便多了。
> 
> 也可以用 Bearer Token 方式，但 Token 会过期，
> 需要定期从浏览器开发者工具重新抓取。

#### 第二步：选择知识库（1:15-1:30）

**画面**：加载知识库列表 → 选择目标知识库

**旁白/字幕**：

> 第二步：选择知识库。
> 
> 点"加载知识库"，你的所有知识库就出来了。
> 选你要同步的那个就行。

**【画面操作实录】**
- 点击"加载知识库"
- 下拉菜单出现知识库列表
- 选择一个知识库

#### 第三步：选择本地目录（1:30-1:50）

**画面**：点击"选择目录" → 文件夹选择弹窗 → 扫描结果

**旁白/字幕**：

> 第三步：选择本地文件夹。
> 
> 工具会自动扫描里面的文件，
> 告诉你找到了多少个文件，
> 不支持的格式也会提前提醒。

**【画面操作实录】**
- 点击"选择目录"
- 选择文件夹
- 显示文件数量和不支持类型的提示

#### 第四步：查看差异 & 一键上传（1:50-2:30）

**画面**：点击"开始对比" → 差异结果 → 一键上传

**旁白/字幕**：

> 第四步：点击"开始对比"。
> 
> **【画面：差异结果展示】**
> 四种状态一目了然：
> - 绿色标签 "仅在本地" — 这些文件 ima 还没有
> - 黄色标签 "仅在 ima" — 这些文件本地缺失
> - 蓝色标签 "内容不同" — 两端版本不一致
> - 灰色标签 "一致" — 已经同步好了
> 
> 看到差异后，点"一键上传所有新文件"，
> 确认后工具就自动把文件传到 ima 了。
> 
> **【画面：上传过程 + 日志输出】**
> 上传完成后，状态标签变成绿色"已上传"。

### 增量同步演示（2:30-2:55）

**旁白/字幕**：

> 日常维护，几秒就知道有没有需要同步的新文件。

**【画面】**：第二次对比结果，大部分文件显示"一致"，只有 1-2 个新文件

## 📋 视频制作备注

### 录屏要点
- 浏览器窗口建议全屏或最大化，避免杂乱背景
- 操作节奏不要太快，每一步停留 2-3 秒让观众看清
- 上传过程可以适当加快，用进度条动画代替等待

### 画面素材建议
- 开场痛点画面：可用桌面文件夹截图 + 手动上传 ima 的操作截图对比
- 差异结果展示：最好有真实数据（不是空结果），展示 5-10 个文件的差异更直观
- 上传成功画面：标签从"仅在本地"变为"已上传"的变化动画

### 字幕/旁白风格
- 口语化、简短、有节奏感
- 避免技术术语堆砌（比如不要说"HMAC-SHA1 签名"，说"用官方接口上传更稳定"）
- 适当用问句引导思考（"你是不是也有这样的烦恼？"）

### BGM 建议
- 轻快的电子/纯音乐，音量低不干扰旁白
- 推荐节奏：开场稍慢 → 演示部分中等 → 收尾轻快

### 可选加分项
- 片尾 5 秒动画 logo
- 操作过程中鼠标高亮圈（屏幕录制软件通常支持）
- 关键步骤加数字标注角标（1️⃣ 2️⃣ 3️⃣ 4️⃣）
