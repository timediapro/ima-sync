# ima 知识库同步工具 — 使用说明

> 让本地文件和 ima 知识库轻松同步，告别手动上传。

---

## 一、它能做什么？

ima 知识库同步工具帮你对比本地文件夹和 ima 知识库的内容差异，并提供一键上传/下载能力。

| 场景 | 工具帮你做的事 |
|------|---------------|
| 初建知识库 | 批量把本地文档传到 ima，不用一个个手动上传 |
| 日常更新 | 增量同步——只传新改过的文件，省时省力 |
| 定期检查 | 生成差异报告，一目了然哪些文件不同步 |
| 下载回本地 | 把 ima 上有但本地缺失的文件拉回来 |

支持 9 种文件类型（PDF、Word、PPT、Excel、Markdown、TXT、图片、Xmind、音频），21 种扩展名。

## 二、三种使用方式

### 方式一：可视化工具（最简单，推荐入门）

**适合人群**：不想写命令行的普通用户

1. 用 Chrome 或 Edge 打开 `index.html`
2. 四步操作，跟着界面走就行

### 方式二：Python 命令行（适合自动化）

**适合人群**：有 Python 环境、想做定时任务的用户

```bash
# 一行命令搞定
python ima_sync_report.py --local "你的目录" --kb 知识库ID
```

### 方式三：WorkBuddy 自动化（最省心）

**适合人群**：想让工具每天自动跑的用户

设置定时任务，每天自动生成差异报告推给你。

## 三、详细操作步骤（可视化工具）

### 前置准备

- 浏览器：Chrome 127+ 或 Edge 127+（其他浏览器可能不支持选择本地目录）
- ima 账号：已登录 ima.qq.com

### 第一步：配置认证

打开 `index.html`，你会看到两个 Tab：

#### 推荐：OpenAPI 方式

1. 访问 [ima.qq.com/agent-interface](https://ima.qq.com/agent-interface)
2. 登录后，创建或查看已有的 Client ID 和 API Key
3. 回到工具界面，填入 Client ID 和 API Key
4. 点击"保存"

> **为什么推荐 OpenAPI？** 稳定、不会过期、不用从浏览器抓 Token。一次填写，长期使用。

#### 备选：Bearer Token 方式

1. 打开 [ima.qq.com](https://ima.qq.com)，登录
2. 按 F12 → Network 标签
3. 在页面上随便操作一下（比如搜索），找到 API 请求
4. 点击请求 → Headers → 复制 `Authorization: Bearer xxx` 中的 `xxx` 部分
5. 粘贴到工具的 Token 输入框
6. 点击"保存"

> **注意**：Bearer Token 会过期，过期后需要重新抓取。建议优先用 OpenAPI。

> **贴心提示**：凭证会自动保存在浏览器 localStorage 中，下次打开工具不用重复输入。

### 第二步：选择知识库

认证保存后，点击"加载知识库"，下拉菜单会出现你所有知识库。选择你想同步的那个。

### 第三步：选择本地目录

点击"选择目录"，在弹窗中选一个文件夹。工具会自动扫描里面的文件，并显示：
- 找到的文件数量
- 不支持的文件类型警告（如果有）

### 第四步：查看差异 & 操作

点击"开始对比"，工具会对比本地和 ima 的文件，显示四种状态：

| 标签 | 含义 | 可操作 |
|------|------|--------|
| 🟢 仅在本地 | 本地有，ima 没有 | 可上传 |
| 🟡 仅在 ima | ima 有，本地没有 | 可下载 |
| 🔵 内容不同 | 两端都有但内容变了 | 可上传更新版 |
| ⚪ 一致 | 两端完全相同 | 无需操作 |

**快捷操作按钮：**
- "选中所有新文件" — 一键勾选所有 🟢 状态的文件
- "选中所有变更文件" — 一键勾选所有 🔵 状态的文件
- "取消全选" — 清除所有勾选

**一键上传：** 勾选后点击"上传选中项"，或直接点击"一键上传所有新文件"按钮（会弹出确认对话框）。

**预检标记：**
- ❌ "不支持" — 文件类型 ima 不接受（如视频文件），自动跳过
- ⚠️ "超限" — 文件超过大小限制，需要压缩后再传

## 四、详细操作步骤（Python 命令行）

### 前置准备

- Python 3.10+ 环境
- 无需安装第三方库（纯标准库实现）

### 设置认证

```bash
# OpenAPI 认证（推荐）
export IMA_OPENAPI_CLIENTID="你的ClientID"
export IMA_OPENAPI_APIKEY="你的APIKey"

# 或者 Bearer Token 认证（备选）
export IMA_TOKEN="你的BearerToken"
```

> Windows 用户用 `set` 代替 `export`：
> ```cmd
> set IMA_OPENAPI_CLIENTID=你的ClientID
> set IMA_OPENAPI_APIKEY=你的APIKey
> ```

### 常用命令

```bash
# 预览模式——只看差异，不上传（新手推荐第一次用这个）
python ima_sync_report.py --local "C:/我的文档" --kb 0019ecbe564011f4 --dry-run

# 增量同步——只上传变化的部分（日常推荐）
python ima_sync_report.py --local "C:/我的文档" --kb 0019ecbe564011f4

# 全量同步——忽略历史记录，从头对比（首次使用推荐）
python ima_sync_report.py --local "C:/我的文档" --kb 0019ecbe564011f4 --full
```

### 参数说明

| 参数 | 必填 | 说明 |
|------|------|------|
| `--local` | ✅ | 本地目录路径 |
| `--kb` | ✅ | 知识库 ID（从 ima 获取或用默认值） |
| `--dry-run` | ❌ | 预览模式，只生成报告不执行上传 |
| `--full` | ❌ | 全量模式，忽略 sync_state.json |

### 输出结果

- HTML 报告：保存在 `scripts/reports/` 目录（最近保留 30 份）
- 同步状态：保存在 `scripts/sync_state.json`（下次增量同步用）

## 六、常见问题

### Q1：认证信息怎么获取？

**OpenAPI（推荐）**：登录 ima.qq.com → 进入 [agent-interface](https://ima.qq.com/agent-interface) → 创建 Client ID 和 API Key → 复制到工具中。

**Bearer Token**：ima.qq.com → F12 → Network → 找到任意 API 请求 → 复制 Authorization header 中 Bearer 后面的部分。

### Q2：知识库 ID 在哪看？

在可视化工具中，加载知识库列表后直接选择即可。如果用命令行，可以通过 ima MCP 工具或 API 获取知识库列表来找到 ID。

### Q3：为什么有些文件标记为"不支持"？

ima API 不支持视频文件的上传。这些文件需要在 ima 桌面客户端中手动上传。

### Q4：增量同步和全量同步有什么区别？

- **增量同步**（默认）：只上传和上次 sync_state.json 记录不同的文件，省时间
- **全量同步**（`--full`）：忽略历史记录，重新对比所有文件，适合首次使用或怀疑记录不准确时

### Q5：sync_state.json 是什么？

记录每次上传成功的文件信息（大小、修改时间）。下次运行增量同步时，只对比和这个记录不同的文件。删除这个文件等同于全量同步。

### Q6：上传失败怎么办？

检查以下几点：
1. 认证信息是否过期（Bearer Token 模式常见）
2. 文件是否超过大小限制
3. 文件名是否包含特殊字符
4. 网络是否正常

如果 COS 直传失败，工具会自动回退到 upload_url 方式。

### Q7：下载文件时为什么有些只能获取文本版？

ima API 的 `fetch_content` 接口返回的是文件文本内容，对 PDF、Word 等二进制文件只能获取文本版本。如需原始文件，请到 ima 桌面端下载。

### Q8：工具支持哪些浏览器？

Chrome 127+ 或 Edge 127+。其他浏览器不支持 File System Access API（选择本地目录的功能），会显示兼容性提示。不支持的浏览器可用 Python CLI 方式。

## 七、进阶玩法

### 每日自动同步检查

通过 WorkBuddy 自动化，设置每天定时运行 `--dry-run` 模式，只看差异不执行上传。发现变化时手动决定是否同步。

### 多知识库同步

分别对不同知识库运行命令，每个知识库有独立的 sync_state.json 记录。

### 大批量首次上传

首次建知识库时用 `--full` 模式，一次把所有文档传上去。后续日常维护用默认增量模式。
